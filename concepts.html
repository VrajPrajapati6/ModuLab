1. Introduction to Digital Communication

Digital communication is the process of transmitting information in binary (digital) form between a sender and receiver through a communication channel. Unlike analog communication which uses continuous signals, digital communication encodes information as discrete values (typically 0s and 1s).

The Digital Communication Pipeline

A complete digital communication system follows a structured pipeline where data passes through several processing stages:

  1. Source Data: The original binary data to be transmitted (e.g., text, image, or sensor data converted to bits).
  2. Line Coding: Converts binary data into electrical signals suitable for transmission over a physical medium.
  3. Digital Modulation: Maps the digital signal onto a high-frequency analog carrier for efficient transmission through channels like air, cable, or fiber.
  4. Channel Transmission: The modulated signal passes through a physical channel where it is degraded by noise and interference.
  5. Error Detection & Correction: Redundancy bits are added before transmission so that errors introduced by the channel can be detected and corrected at the receiver.
  6. Multiplexing: Multiple signals are combined into a composite signal for efficient channel utilization.
Why Digital? Digital communication offers significant advantages over analog: better noise immunity, easier encryption, ability to detect and correct errors, and efficient compression. These benefits have made digital communication the dominant paradigm in modern systems.

2. Line Coding Techniques

Line coding converts a sequence of binary digits (bits) into a digital signal. The resulting signal determines the voltage levels, timing, and synchronization characteristics of the transmitted waveform.

NRZ-L (Non-Return-to-Zero Level)

The voltage level directly represents the bit value. Bit 1 is a positive voltage, bit 0 is a negative voltage. The signal does not return to zero between bits.

Bit 1 → +V , Bit 0 → −V

NRZ-I (Non-Return-to-Zero Inverted)

The voltage level changes (inverts) at the start of a bit period for a "1", and stays the same for a "0". This provides better clock recovery than NRZ-L.

Bit 1 → Invert , Bit 0 → No Change

Manchester Encoding

Each bit period is divided into two halves. A transition occurs in the middle of every bit period. Bit 1 is high-to-low, bit 0 is low-to-high. Guarantees clock synchronization at the cost of doubling the bandwidth.

Bit 1 → High→Low , Bit 0 → Low→High

Differential Manchester Encoding

Always has a mid-bit transition (for clocking). The presence or absence of a transition at the start of each bit period encodes the data. A transition at the start means 0; no transition at the start means 1.

AMI (Alternate Mark Inversion)

Zero bits are represented by 0V. One bits alternate between +V and −V. This bipolar scheme provides built-in error detection (a bipolar violation occurs if two consecutive 1s have the same polarity).

Bit 0 → 0V , Bit 1 → alternates +V / −V

3. Digital Modulation

Modulation maps a digital signal onto a high-frequency analog carrier wave for efficient transmission. The carrier wave has three properties that can be varied: amplitude, frequency, and phase.

Carrier: s(t) = A · sin(2πf·t + φ)

ASK (Amplitude Shift Keying)

The amplitude of the carrier is varied to represent data. Bit 1 transmits the carrier at full amplitude; bit 0 transmits at zero amplitude (on-off keying).

s(t) = A·m(t)·sin(2πfc·t)

where m(t) = 1 for bit 1, m(t) = 0 for bit 0.

FSK (Frequency Shift Keying)

The frequency of the carrier is shifted between two values. Bit 1 uses a higher frequency f₁, and bit 0 uses a lower frequency f₂.

s(t) = A·sin(2πf₁·t) for bit 1 , A·sin(2πf₂·t) for bit 0
Continuous Phase FSK: In practical FSK, the phase is maintained continuously across bit transitions to avoid abrupt discontinuities, which would cause spectral spreading.

PSK (Phase Shift Keying)

The phase of the carrier is shifted to represent data. In Binary PSK (BPSK), bit 1 has phase 0° and bit 0 has phase 180°.

s(t) = A·sin(2πfc·t + 0) for bit 1 , A·sin(2πfc·t + π) for bit 0

4. Channel & Noise

In any real communication system, the transmitted signal is degraded as it passes through the physical channel. The most common model for channel noise is AWGN (Additive White Gaussian Noise).

r(t) = s(t) + n(t)

where:
  r(t) = received signal
  s(t) = original transmitted signal
  n(t) = Gaussian noise with mean 0 and standard deviation σ

Properties of AWGN

  • Additive: The noise adds linearly to the signal.
  • White: It has uniform power spectral density across all frequencies.
  • Gaussian: The amplitude of the noise follows a Gaussian (normal) distribution.

Signal-to-Noise Ratio (SNR)

The quality of a received signal is measured by the ratio of signal power to noise power:

SNR = P_signal / P_noise    (often expressed in dB: SNR_dB = 10·log₁₀(SNR))
Box-Muller Transform: The simulator generates Gaussian noise using the Box-Muller transform, which converts uniformly distributed random numbers into Gaussian-distributed values:

z = √(−2·ln(u₁)) · cos(2π·u₂) , where u₁, u₂ ∈ (0, 1)

5. Error Detection & Correction

Errors occur when noise flips bits during transmission. Error control techniques add redundancy to detect or correct these errors.

Parity Check

A single parity bit is appended to the data. In even parity, the parity bit is set so that the total number of 1s (including the parity bit) is even. In odd parity, the total is odd.

Even Parity: count(1s in data + parity bit) mod 2 = 0

Limitation: Can only detect an odd number of bit errors. Cannot correct errors.

CRC (Cyclic Redundancy Check)

A polynomial division-based approach. The data bits are treated as a polynomial and divided by a generator polynomial. The remainder is appended to the data as the CRC.

Generator: x³ + x + 1 → binary 1011

Steps:

  1. Augment data with k zeros (k = degree of generator polynomial)
  2. Perform XOR-based polynomial division
  3. Append the remainder to original data
  4. At receiver, divide received data by same generator — remainder 0 means no error

Hamming Code (7,4)

A forward error correction code that encodes 4 data bits into 7 bits (adding 3 parity bits). It can detect up to 2-bit errors and correct all single-bit errors.

Codeword: [p1, p2, d1, p3, d2, d3, d4]

Parity bit positions (1, 2, 4) cover specific data bit combinations:

  • p1 covers positions 1, 3, 5, 7 → p1 = d1 ⊕ d2 ⊕ d4
  • p2 covers positions 2, 3, 6, 7 → p2 = d1 ⊕ d3 ⊕ d4
  • p3 covers positions 4, 5, 6, 7 → p3 = d2 ⊕ d3 ⊕ d4
Syndrome Decoding: At the receiver, the syndrome is computed as [s3, s2, s1]. If the syndrome is non-zero, its decimal value indicates the 1-indexed position of the error — which can then be corrected by flipping that bit.

6. Multiplexing

Multiplexing allows multiple data streams to share a single communication channel, improving efficiency and reducing cost.

TDM (Time Division Multiplexing)

Each signal is allocated a specific time slot in a repeating frame. Bits from different sources are interleaved in round-robin fashion.

Frame: [S1-bit1, S2-bit1, S1-bit2, S2-bit2, ...]

FDM (Frequency Division Multiplexing)

Each signal is modulated onto a different carrier frequency. All signals are transmitted simultaneously but occupy different frequency bands.

Combined(t) = S1(t)·sin(2πf₁t) + S2(t)·sin(2πf₂t)
Key Difference: TDM divides the channel in the time domain (each signal gets a time slot), while FDM divides it in the frequency domain (each signal gets a frequency band). TDM is simpler for digital signals; FDM is traditional for analog systems like radio.

7. Summary

A complete digital communication system transforms raw binary data through a series of processing stages to ensure reliable, efficient transmission:

  1. Line Coding converts bits into voltage-level signals with desired properties (self-clocking, no DC component).
  2. Modulation maps the digital signal onto a carrier wave for transmission over physical media.
  3. The channel introduces noise (modeled as AWGN), degrading the signal.
  4. Error detection/correction techniques (Parity, CRC, Hamming) add redundancy to combat noise-induced errors.
  5. Multiplexing (TDM, FDM) combines multiple signals for efficient shared channel usage.
